Installation

Introduction

This document will explain how to configure your own ShiftSpace server. This will enable you to develop your own Spaces or hack on the core platform itself. If you have any difficulties along the way, please consult one of our email discussion lists. We have one for users as well as one for developers. If you just want to install ShiftSpace (the client userscript) you don't need to follow these directions, just go to ShiftSpace.org and click the install button.


The Installation Guides

We currently have installation guides of OS X, Windows XP, and Ubuntu. If you have installation instructions for other operating systems, please let us know. If you encounter errors or issues feel free to contact us.


Notes for Developers

Currently the best supported development environment for ShiftSpace is Firefox 3.5 with following extensions installed:

  • FireBug 1.5x.0a24
  • The latest version of GreaseMonkey

Optional

What follows are a variety of optional installs for the ShiftSpace development environment. These aren't strictly required for you start building your space but are useful for testing how your space might work out in the wild wild web.


Proxy

If you would like to ensure that your space works properly under the proxy so that users who are not running the ShiftSpace plugin can see your space's shifts, you should install the proxy.

First you need to install libxml2 and libxslt2 as appropiate for you platform. As these are well covered elsewhere we won't go over them here.

We assume that you've already installed Python 2.4 or greater as well as setuptools (not required on OS X 10.5 or 10.6). If so you can run the following from a terminal:

$ sudo python easy_install lxml
